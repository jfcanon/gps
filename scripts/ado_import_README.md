# ADO Import — MCSB v2 User Stories

Imports ~160 MCSB v2 user stories into Azure DevOps as Features + User Stories via REST API.

---

## Prerequisites

- Python 3.11+
- `requests` and `python-dotenv` (already in `requirements.txt`)
- ADO Personal Access Token (PAT) with **Work Items: Read, Write, Manage** scope

Install dependencies:
```bash
cd /path/to/NewSecGap
pip install -r scripts/requirements.txt
```

---

## Step 1 — Configure

Open `scripts/ado_config.py` and edit:

```python
ADO_ORG     = "https://dev.azure.com/YOUR_ORG"   # your org URL
ADO_PROJECT = "YOUR_PROJECT"                       # your project name
AREA_PATH       = None   # optional, e.g. "ProjectName\\Security"
ITERATION_PATH  = None   # optional, e.g. "ProjectName\\Sprint 1"
```

Set your PAT as an environment variable — **never paste it into the file**:

```bash
export ADO_PAT=<your-personal-access-token>
```

---

## Step 2 — Parse .md files → CSV

```bash
cd scripts/
python parse_stories.py
```

Output: `scripts/ado_stories.csv` — 160 rows, one per user story.

Verify:
```bash
wc -l ado_stories.csv          # expect 161 (header + 160 rows)
head -2 ado_stories.csv        # check columns
```

## Step 4 — Import

```bash
python import_to_ado.py
```

What happens:
1. **Pass 1** — Creates 12 Features (one per domain). Skips any that already exist (found by title search).
2. **Pass 2** — Creates ~160 User Stories, each linked to its parent Feature.

Expected output:
```
── Pass 1: Features ──
    Created Feature #1001: [SEC-NS] Network Security — MCSB v2
    ...
── Pass 2: User Stories (160 total) ──
    1. Created #1013 — NS SEC-1: [SEC-1] Implement Azure DDoS Protection Standard...
    ...
IMPORT COMPLETE
  Features:     12
  Stories OK:   160
  Stories FAIL: 0
```

---

## Re-running safely

- **Features**: idempotent — existing Features are found by title search, not duplicated.
- **User Stories**: NOT idempotent — re-running creates duplicates. Check ADO before re-running.

To avoid duplicates on retry after partial failure:
1. Delete the partial import in ADO first, **or**
2. Manually note which story_ids succeeded and remove those rows from `ado_stories.csv` before re-running.

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `ADO_PAT environment variable not set` | Missing env var | `export ADO_PAT=<pat>` |
| `ADO_ORG not configured` | Default value in config | Edit `ado_config.py` |
| `401 Unauthorized` | PAT wrong or expired | Regenerate PAT in ADO User Settings |
| `403 Forbidden` | PAT lacks Work Items scope | Regenerate with correct scope |
| `404 Not Found` | Wrong org URL or project name | Verify `ADO_ORG` and `ADO_PROJECT` |
| `TF401232` | Area/Iteration path does not exist | Create the path in ADO first, or set `AREA_PATH = None` |
| Stories FAIL > 0 | Rate limit or network | Re-run; Pass 1 is safe, Pass 2 duplicates — delete partial first |

## CSV Schema Reference

| Column | Example | Notes |
|---|---|---|
| `domain` | `NS` | 2-letter domain code |
| `feature_name` | `[SEC-NS] Network Security — MCSB v2` | ADO Feature title |
| `story_id` | `SEC-1` | Control number within domain |
| `story_title` | `[SEC-1] Implement Azure DDoS...` | ADO User Story title |
| `resource` | `Virtual Network` | v3 resource (empty for pure v2) |
| `is_combined` | `True` | True = v2+v3 combined story |
| `description` | `As a cloud engineer...` | Full As a / I want / so that text |
| `acceptance_criteria` | `NS-1 config assessed...` | 3 bullets, newline-separated |
| `tags` | `Security` | ADO tag |
